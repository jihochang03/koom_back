# KOOM 화면 명세서

> 화면별로 **진입 조건 · 조회/처리 API · 표시 데이터(필드) · 입력 데이터(필드) · 동작 · 상태 전이**를 실제 백엔드 모델/엔드포인트 기준으로 명세합니다.
> 시각화: [화면명세서.html](./화면명세서.html) · 관련: [IA 정보구조](./IA_정보구조.md) · [요구사항 추적표](./요구사항추적표.md)

표기 규칙
- **표시** = 화면에 출력되는 읽기 데이터 / **입력** = 사용자가 입력·전송하는 데이터
- 각 데이터는 **자연어 설명(디자이너용)** + **변수명(백엔드용 실제 모델/시리얼라이저 필드)** 으로 함께 표기. `*` = 필수
- 액터: 고객 / CS / 본사 / 시스템
- 시각화 HTML([화면명세서.html](./화면명세서.html))은 화면별로 "한글 설명 — `변수`" 형태로 한 줄씩 정리되어 있습니다.
