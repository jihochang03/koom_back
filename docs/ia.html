KOOM 정보구조 (IA) 플로우차트

각 화면은 단 하나의 노드로만 존재하며, 화살표가 화면 전환을 나타냅니다. 여러 곳에서 들어오는 공통 화면(S-04 상품상세, S-10 주문상세)은 화살표가 한 노드로 수렴하고, 나가는 출구는 항상 동일합니다.

고객 CS 본사 공개 진입/시스템 ⇲ 수렴 여러 화면이 한 노드로
고객 앱CS 콘솔본사 어드민

고객 앱 진입 → 탐색 → 상품 → 구매 → 주문·사후

앱 실행 시 토큰 유무로 분기 → 로그인 → 홈(허브)에서 갈라짐. URL입력·상품목록·쇼핑몰 3경로가 모두 상품상세(S-04) 한 노드로 수렴 → 담기 → 결제 → 완료 → 상세. 주문완료·주문목록은 주문상세(S-10) 한 노드로 수렴.

CS 콘솔 대리구매 → 검수 → FastBox (process 체인)

담당 추적(허브)에서 각 작업으로 진입. 핵심은 대리구매 → 검수 → FastBox 인계로 이어지는 처리 순서이며, 각 단계가 주문 상태를 전이시킵니다.

본사 어드민 모니터링 허브 → 관리 영역

실시간 모니터링(허브)에서 상품·정산·설정·감독 영역으로 분기합니다.
